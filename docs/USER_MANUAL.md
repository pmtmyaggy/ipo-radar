## 一、产品概述

### 1.1 什么是 IPO-Radar？

IPO-Radar 是一款面向美股投资者的 IPO 信息聚合与决策辅助工具。它自动追踪即将上市的新股，整合多维度数据（招股书、市场情绪、基本面、机构持仓），输出可量化的投资决策评分，帮助投资者在 IPO 窗口期做出更明智的选择。

### 1.2 核心价值

- **信息聚合**: 整合 SEC EDGAR、Yahoo Finance、Nasdaq 等多源数据
- **智能评分**: 5 维度量化评分系统（基本面、市场热度、行业前景、估值、机构背书）
- **实时监控**: 自动追踪 IPO 日历、价格变动、财报发布、禁售期到期
- **通知推送**: 关键事件自动推送到飞书/钉钉

### 1.3 适用人群

| 用户类型 | 使用场景 |
|---------|---------|
| **短线交易者** | 寻找首日有爆发力的新股，快速进出 |
| **价值投资者** | 筛选优质成长股，长期持有 |
| **机构投资者** | 批量跟踪 IPO Pipeline，生成研究报告 |
| **量化研究员** | 获取结构化 IPO 数据，构建策略 |

## 三、安装与启动

### 3.1 环境要求

- **操作系统**: macOS / Linux / Windows (WSL)
- **Python**: 3.10+
- **内存**: 最低 4GB，推荐 8GB
- **磁盘**: 最低 1GB 可用空间

### 3.2 安装步骤

#### 方式一: 本地安装

```bash
# 1. 克隆代码库
git clone https://github.com/pmtmyaggy/ipo-radar.git
cd ipo-radar

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填写必要配置

# 4. 启动 Dashboard
make dashboard
```

#### 方式二: Docker 安装

```bash
# 1. 构建镜像
make docker-build

# 2. 启动服务
make start

# 3. 查看状态
make docker-status
```

### 3.3 访问系统

启动成功后，在浏览器中访问:

```
http://localhost:8501
```

## 四、功能模块详解

### 4.1 IPO 日历

**位置**: 左侧导航栏 → 📅 IPO日历

**功能**: 展示未来即将上市的 IPO 事件

**显示字段**:
- 股票代码 (Ticker)
- 公司名称 (Company Name)
- 预计上市日期 (Expected Date)
- 发行价格区间 (Price Range)
- 发行股数 (Shares Offered)
- 主承销商 (Lead Underwriter)

**使用场景**: 
- 提前锁定关注标的
- 规划申购策略

**数据来源**: Nasdaq API

---

### 4.2 股票列表

**位置**: 左侧导航栏 → 📈 股票列表

**功能**: 查看系统中跟踪的所有 IPO 股票

**显示字段**:
- 股票代码
- 公司名称
- 上市天数
- 当前价格 vs IPO 价格
- 综合信号
- 基本面评分

**排序功能**: 可按综合评分、上市天数、涨跌幅排序

**筛选功能**: 可按状态（即将上市/已上市/观察中）筛选

---

### 4.3 个股详情页

**进入方式**: 点击股票列表中的任意股票，或在搜索框输入股票代码

#### 4.3.1 基本信息卡片

显示四个核心指标：

| 指标 | 说明 |
|------|------|
| **当前价格** | 最新收盘价 |
| **vs IPO** | 相对于 IPO 发行价的涨跌幅 |
| **上市天数** | 距离 IPO 日的天数 |
| **基本面评分** | 0-100 分的综合基本面评分 |

#### 4.3.2 价格走势图

- K 线图展示从 IPO 日到现在的价格走势
- 标注关键事件点（财报、禁售期到期等）

#### 4.3.3 四窗口状态

IPO 投资的四个关键时间窗口：

| 窗口 | 时间范围 | 说明 |
|------|---------|------|
| **首日回调** | IPO 后 1-5 天 | 观察首日炒作后的回调机会 |
| **底部突破** | IPO 后 20-60 天 | 股价筑底后的突破信号 |
| **禁售期** | IPO 后 90-180 天 | 内部人股票解禁，供应冲击 |
| **首次财报** | IPO 后 60-150 天 | 首份季报发布，业绩验证 |

**状态说明**:
- 🟢 **活跃**: 当前处于该窗口期内
- 🔴 **已过期**: 窗口期已结束
- ⚪ **未激活**: 条件不满足

#### 4.3.4 综合判断

基于多维度数据生成的投资建议：

**信号类型**:
- **STRONG_OPPORTUNITY** (强烈推荐): 多个强信号叠加
- **OPPORTUNITY** (值得关注): 有明显机会信号
- **WATCH** (观察中): 正在形成机会
- **NO_ACTION** (无操作): 暂无明确信号
- **RISK** (风险提示): 存在明显风险

**信号原因**: 列出触发信号的具体原因

**风险因素**: 列出需要注意的风险点

### 4.4 S-1 文件

**位置**: 左侧导航栏 → 📄 S-1文件

**功能**: 查看 IPO 招股书（Form S-1）

**显示内容**:
- 公司基本信息
- 财务数据摘要
- 募集资金用途
- 风险因素
- 管理层信息

**使用场景**:
- 深度研究公司基本面
- 了解风险因素
- 查看股权结构

---

### 4.5 基本面分析

**位置**: 左侧导航栏 → 💰 基本面

**评分维度** (总分 100 分):

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **营收增长** | 25% | >20%: 25分, 10-20%: 15分, <10%: 0分 |
| **毛利率** | 20% | >60%: 20分, 40-60%: 12分, <40%: 0分 |
| **现金跑道** | 20% | >18月: 20分, 12-18月: 10分, <12月: 0分 |
| **债务水平** | 15% | 债务/资产<0.3: 15分, 0.3-0.6: 8分, >0.6: 0分 |
| **承销商等级** | 10% | 前十投行: 10分, 其他: 5分 |
| **市值规模** | 10% | >$1B: 10分, $500M-1B: 5分, <$500M: 0分 |

**评分等级**:
- **80-100**: 优秀 (强烈推荐)
- **60-79**: 良好 (值得关注)
- **40-59**: 一般 (中性观望)
- **0-39**: 较差 (建议回避)

---

### 4.6 综合评分

**位置**: 左侧导航栏 → 📊 综合评分

**评分维度** (总分 100 分):

| 维度 | 权重 | 说明 |
|------|------|------|
| **基本面** | 30% | 财务健康度、盈利能力 |
| **市场热度** | 25% | 认购倍数、社交媒体讨论量 |
| **行业前景** | 20% | 行业增速、竞争格局 |
| **估值水平** | 15% | 定价区间合理性 |
| **机构背书** | 10% | 承销商等级、基石投资者 |

**使用建议**:
- **80+ 分**: 优质标的，重点关注
- **60-79 分**: 一般标的，结合其他因素判断
- **< 60 分**: 谨慎对待

---

### 4.7 机构持仓

**位置**: 左侧导航栏 → 🏦 机构持仓

**功能**: 查看 13F 机构持仓数据

**显示内容**:
- 新进机构名单
- 退出机构名单
- 持仓变化排名
- 前十大股东

**更新频率**: 季度更新（13F 每季度披露一次）

**使用场景**:
- 跟随"聪明钱"建仓
- 验证市场认可度

## 五、使用场景示例

### 场景一: 短线交易 IPO

**目标**: 寻找首日有爆发力的新股

**操作步骤**:

1. **盘前准备** (IPO 前一天)
   ```
   打开 Dashboard → IPO日历
   查看明日上市的新股列表
   关注定价区间和超额认购倍数
   ```

2. **基本面初筛**
   ```
   点击感兴趣的新股
   查看基本面评分（>60分值得关注）
   查看行业前景和市场热度
   ```

3. **开盘交易** (IPO 当天)
   ```
   观察开盘价相对于定价的涨幅
   如果涨幅 <30%，可能存在机会
   结合市场情绪判断是否入场
   ```

4. **盘后分析**
   ```
   查看首日表现数据
   更新观察清单
   ```

### 场景二: 价值投资筛选

**目标**: 寻找长期持有的优质 IPO

**操作步骤**:

1. **批量筛选**
   ```
   股票列表 → 筛选基本面评分 >70
   按行业分类查看
   ```

2. **深度研究**
   ```
   逐个查看 S-1 文件
   重点关注:
   - 营收增长趋势
   - 毛利率水平
   - 市场竞争格局
   - 管理层背景
   ```

3. **上市后跟踪** (IPO 后 3-6 个月)
   ```
   观察价格走势
   等待首个财报发布
   查看机构持仓变化
   ```

4. **建仓时机**
   ```
   等待四窗口信号出现
   基本面评分保持高位
   机构开始建仓
   ```

### 场景三: 禁售期套利

**目标**: 利用禁售期到期前后的价格波动

**操作步骤**:

1. **识别标的**
   ```
   股票列表 → 筛选上市 150-210 天的股票
   查看锁定期到期日
   ```

2. **分析供应冲击**
   ```
   个股详情 → 查看禁售期信息
   关注锁定股份比例
   估算潜在抛压
   ```

3. **制定策略**
   ```
   如果锁定比例 >20%: 禁售期前可能下跌，到期后可能反弹
   如果锁定比例 <10%: 影响较小
   ```

4. **执行交易**
   ```
   禁售期前 1-2 周减仓或做空
   到期后观察市场反应
   如有反弹迹象，逐步建仓
   ```

## 六、常见问题

### Q1: 为什么某些股票显示 "N/A"？

**原因**:
- 该股票不在数据库中
- 数据抓取失败
- 该字段不适用（如未上市的 IPO 没有"vs IPO"数据）

**解决**:
- 等待系统自动更新
- 手动运行 `make update` 更新数据

### Q2: 基本面评分为什么是 0？

**原因**:
- 无法获取财务数据
- S-1 文件未解析
- 已上市股票缺乏历史数据

**解决**:
- 检查网络连接
- 对于已上市股票，评分可能不准确，建议参考其他指标

### Q3: 飞书通知没有收到？

**检查清单**:
- [ ] `.env` 中 `FEISHU_WEBHOOK_URL` 已配置
- [ ] Webhook URL 格式正确
- [ ] 机器人已添加到群聊
- [ ] 群聊设置中启用了该机器人

**测试命令**:
```bash
python -c "from src.notifier import FeishuNotifier; FeishuNotifier().send_message('测试', '配置成功')"
```

### Q4: 数据多久更新一次？

| 数据类型 | 更新频率 |
|---------|---------|
| 股价数据 | 实时 (15分钟延迟) |
| IPO日历 | 每日 |
| S-1文件 | 每小时扫描 |
| 机构持仓 | 季度 (13F披露后) |
| 基本面评分 | 每日 |

### Q5: 可以跟踪港股/A股 IPO 吗？

**当前版本**: 仅支持美股 IPO

**未来计划**: 考虑扩展至港股

### Q6: 如何添加自定义股票到观察清单？

```python
from src.radar.monitor import IPORadar

radar = IPORadar()
radar.add_to_watchlist('AAPL', ipo_date=date(2020, 12, 12))
```

## 七、故障排查

### 7.1 Dashboard 无法启动

**症状**: 访问 http://localhost:8501 显示错误

**排查步骤**:

1. 检查端口占用
   ```bash
   lsof -i :8501
   ```

2. 检查 Python 环境
   ```bash
   python --version  # 需要 3.10+
   pip list | grep streamlit
   ```

3. 查看日志
   ```bash
   tail -f /tmp/streamlit.log
   ```

### 7.2 数据抓取失败

**症状**: 股票列表为空，或显示 "获取数据失败"

**排查步骤**:

1. 检查网络连接
   ```bash
   curl https://api.nasdaq.com/api/ipo/calendar
   ```

2. 检查 EDGAR 配置
   ```bash
   echo $EDGAR_IDENTITY  # 必须设置
   ```

3. 手动运行更新
   ```bash
   python -c "from src.crawler.api import CrawlerAPI; CrawlerAPI().refresh_ipo_calendar()"
   ```

### 7.3 数据库错误

**症状**: "unable to open database file"

**解决**:

```bash
# 创建数据目录
mkdir -p data
chmod 755 data

# 初始化数据库
python -c "from src.crawler.models.database import init_database; init_database()"
```

### 7.4 性能问题

**症状**: Dashboard 加载缓慢

**优化建议**:

1. 使用 PostgreSQL 替代 SQLite（生产环境）
2. 增加缓存时间
3. 限制同时显示的股票数量
